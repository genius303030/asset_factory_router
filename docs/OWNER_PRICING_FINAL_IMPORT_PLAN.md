# Owner Pricing Final Import Plan

This document is a planning artifact for a possible future owner pricing final
import. G1-025 does not implement production import, does not add a production
write command, and does not mutate live JSON or production pricing data.

## Future Final Import Scope

A future final import would take an owner-approved sandbox pricing output and
write it to an explicit production pricing target after all safety gates pass.
It would be the first workflow stage allowed to create a production pricing
change, so it must be handled as a separate G0-approved task with backup,
validation, rollback, and audit requirements.

The current safe workflow remains:

1. Dry-run preview.
2. Sandbox apply plan.
3. Sandbox apply output.
4. Owner approval gate.
5. Final import planning.

## What G1-025 Does Not Do

- It does not add `asset-factory owner-pricing-final-import`.
- It does not add any production write command.
- It does not mutate live JSON.
- It does not mutate production pricing data.
- It does not weaken sandbox output or approval gate checks.
- It does not approve any real owner pricing data.

## Required Prerequisites Before Future Import

- G0 explicitly approves a future final import implementation task.
- Owner pricing CSV has passed dry-run validation.
- Dry-run report exists and has been reviewed.
- Sandbox apply plan exists and has been reviewed.
- Sandbox output JSON exists and has been reviewed.
- Owner approval record JSON exists.
- Approval record checksum matches the sandbox output JSON bytes.
- Production target path is explicitly identified.
- Backup output path is explicitly identified.
- Rollback procedure has been reviewed by G0 and G2.
- Working tree is clean except for the planned import artifacts.
- No real owner CSV or private pricing data is committed.

## Required Owner Approval Record

The future import must require the G1-024 approval record generated by
`owner-pricing-approve-sandbox-output`.

The approval record must show:

- `approval_type: owner_pricing_sandbox_output_approval`.
- `sandbox_output.sha256` populated.
- `final_import_enabled: false`.
- `live_json_mutated: false`.
- `production_pricing_mutated: false`.
- Approval phrase used: `I_APPROVE_SANDBOX_PRICING_OUTPUT`.
- Approved by field populated.
- Generated timestamp present.

The future import must reject stale or mismatched approval records. A record is
stale if its sandbox output path or checksum does not match the sandbox output
being imported, or if G0 sets a project-specific expiration rule before import.

## Sandbox Output Checksum Matching

Before any future production write:

1. Read the sandbox output JSON as bytes.
2. Compute SHA-256 over those exact bytes.
3. Compare the computed hash with `approval_record.sandbox_output.sha256`.
4. Abort if the hash differs.
5. Abort if the approval record references a different sandbox output path
   unless G0 explicitly approves path relocation and the checksum still matches.

Checksum mismatch means the sandbox output is not the file the owner approved.

## Required Backup Before Import

Before future import writes production pricing:

- The production target must exist unless G0 explicitly approves first-time
  creation.
- The import must copy the current production target to an explicit backup path.
- Backup path must be outside live production paths.
- Backup creation must happen before any production write.
- Backup must include timestamp, source production path, target backup path, and
  checksum.
- Import must abort if backup creation or backup verification fails.

## Required Validation Before Import

Future import must validate:

- Sandbox output JSON parses successfully.
- Sandbox output declares `sandbox_only: true`.
- Sandbox output declares `final_import_enabled: false`.
- Sandbox output declares `live_json_mutated: false`.
- Sandbox output declares `production_pricing_mutated: false`.
- Approval record exists and matches checksum.
- Production target path is explicit and allowed.
- Backup output path is explicit and allowed.
- Production target schema is readable.
- Sandbox output materials have unique `material_key` values.
- No skipped invalid rows are present in the materials list.
- Expected add, update, unchanged, and retained counts match the sandbox summary.
- No unrelated dirty files exist in the repository.

## Required Validation After Import

After a future import writes production pricing, it must validate:

- Production target exists.
- Production target is valid JSON or CSV according to the chosen target format.
- Production pricing schema is valid.
- Material count matches expected imported output.
- Added, updated, unchanged, and retained counts match the approved sandbox
  output summary.
- No duplicate `material_key` values exist.
- No invalid rows were imported.
- Production target checksum is recorded.
- Backup file still exists and its checksum still matches the pre-import
  backup metadata.
- Owner receives a final report with import result, backup path, checksums, and
  rollback instructions.

## Rollback Strategy

Rollback must restore the exact backup created immediately before import. The
future import workflow must not rely on regenerating old pricing from owner CSV
or from memory.

Rollback is required when:

- Import partially fails.
- Post-import schema validation fails.
- Material counts do not match expected output.
- Duplicate material keys are detected after import.
- Owner rejects the final report.
- G0 declares emergency stop.

Detailed rollback procedure lives in
`docs/OWNER_PRICING_FINAL_IMPORT_ROLLBACK_PLAN.md`.

## Emergency Stop Conditions

Abort before production write if any of these occur:

- Approval record missing.
- Approval checksum mismatch.
- Sandbox output missing or invalid.
- Sandbox output does not declare `sandbox_only: true`.
- Sandbox output declares `final_import_enabled: true`.
- Backup cannot be created.
- Backup checksum cannot be verified.
- Production target path is missing or unsafe.
- Repository has unrelated dirty files.
- Duplicate material keys are detected.
- Any skipped invalid row appears in materials.
- Owner or G0 withdraws approval.

Abort after production write and start rollback review if any of these occur:

- Production target fails schema validation.
- Material counts differ from approved sandbox output.
- Output checksum cannot be recorded.
- Final report cannot be written.

## Audit Log Requirements

The future import must write an audit log containing:

- Generated timestamp.
- Command name and version.
- Operator identity or local owner label.
- Production target path.
- Backup path.
- Sandbox output path.
- Sandbox output SHA-256.
- Approval record path.
- Approval record SHA-256.
- Pre-import production checksum.
- Post-import production checksum.
- Counts for added, updated, unchanged, retained, skipped, and duplicate rows.
- Validation results.
- Rollback status.
- Final owner sign-off status.

Audit logs created from real owner data must stay in ignored local paths such as
`output/owner_pricing/`, `outputs/owner_pricing/`, or `owner_pricing_private/`.

## Manual Owner Sign-off Procedure

Before future import:

1. Owner reviews dry-run report.
2. Owner reviews sandbox apply plan.
3. Owner reviews sandbox output JSON.
4. Owner generates approval record with the explicit approval phrase.
5. Owner confirms approval record checksum matches sandbox output.
6. Owner confirms backup path and production target path.
7. Owner gives a separate final import confirmation phrase only after G0
   approves the future import implementation.

After future import:

1. Owner receives final import report.
2. Owner reviews production target checksum, material counts, and validation
   results.
3. Owner confirms backup path is retained.
4. Owner either signs off or requests rollback review.

## Future Command Design

The future command name should be:

```bash
asset-factory owner-pricing-final-import
```

This command is not implemented in G1-025.

Required future inputs:

- `--sandbox-output <sandbox-output.json>`
- `--approval-record <approval-record.json>`
- `--production-target <production-pricing.json-or-csv>`
- `--backup-output <backup-pricing.json-or-csv>`
- `--final-confirmation <exact-final-confirmation-phrase>`
- `--enable-production-import`

Required future safety gates:

- Approval record exists.
- Approval checksum matches sandbox output.
- Sandbox output has `sandbox_only: true`.
- Sandbox output has `final_import_enabled: false`.
- Sandbox output has `live_json_mutated: false`.
- Sandbox output has `production_pricing_mutated: false`.
- Production target path is explicit.
- Backup path is explicit.
- Backup is written before import.
- Import fails if backup cannot be created.
- Production write fails if the repository is dirty.
- Production write fails if target path is unsafe.
- Final confirmation phrase is required.
- `--enable-production-import` is required.

## G1-025 Boundary

G1-025 stops at planning. Any final import implementation must be a separate
task with new tests, G2 review, G0 approval, and a clear production rollback
plan.
