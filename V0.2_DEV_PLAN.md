# V0.2 開發計畫 (V0.2 DEV PLAN)

## 本輪開發目標
將本專案從「可跑的 MVP」升級為「可長期維護、可擴充」的素材工廠核心系統，具備完善的組態設定、驗證機制與多樣的 CLI 檢測工具。

## 本輪新增與升級模組
1. **Python Package & CLI**: 完善 `asset-factory`，新增 `validate`, `inspect`, `list-providers` 等指令。
2. **Configurable Router**: 評分機制改為由外部 JSON 組態 (`config/`) 驅動，包含 8 大維度評分，並輸出詳細的分數細節。
3. **Provider Registry**: 引入 Registry 模式，並詳細定義 Mock Provider 的能力矩陣與風險評估。
4. **Prompt Engine 增強**: 將 Prompt Pack 輸出擴充為 11 個標準段落，並使用獨立的 Markdown 樣板。
5. **Asset Pack 擴充**: 從 7 檔案增加至 10 檔案，包含任務摘要與驗收清單。
6. **Brief 驗證器**: 引入 schema 與 validation 機制，確保輸入的專案需求合法。
7. **測試與檢查機制**: 單元測試擴展至 20 個以上，並新增 Inspect CLI 針對生成結果進行完整性驗證。

## 本輪「不」做哪些事
- 不串接任何真實外部 API（繼續使用 Mock Providers）。
- 不實作短影音實際剪輯或生成。
- 不引入大型 Web 框架（無 UI 介面，純 CLI/Core）。
- 不要求 API Key，也不讀取客戶機密資料。
- 絕不破壞原先 V0.1.1 的概念與使用流程。
